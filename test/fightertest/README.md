# Fightertest sync test

A seeded Beyond All Reason "fightertest" used as the engine's first CI sync test. The `.github/workflows/fightertest.yml` workflow runs this headlessly on top of a pinned BAR game + map version and asserts that a deterministic sync-hash file lands in `$SPRING_DATADIR` at the end of the run. Background github issues: beyond-all-reason/RecoilEngine#2910, parent initiative beyond-all-reason/RecoilEngine#2906.

## Running locally

Build the engine, make sure `spring-headless` and `pr-downloader` are on your PATH or referenced by absolute path, then:

```
pr-downloader --filesystem-writepath ./bar-data \
    --download-game "Beyond All Reason test-29660-e79286c" \
    --download-map  "Jade Empress 1.3"

spring-headless --isolation --write-dir /path/to/bar-data test/fightertest/startscript.txt
```

A successful run exits 0 at game frame 2025 (the startscript's trailing `quitforce`) and leaves a sync-hash file in `$SPRING_DATADIR` for CI to pick up.

## Bumping the pinned BAR version

BAR's rapid `test-<N>-<sha>` tags are content-addressed — once published they never change — which is why the CI doesn't need an external version manifest. To move to a new pin you must edit **two** places in the same commit, or the cache will hand you stale data:

1. `gametype=` and `mapname=` lines in `startscript.txt`
2. The `key:` on the `Restore bar-data cache` step in `.github/workflows/fightertest.yml`

Encoding the game/map identifiers into the cache key is deliberate: a stale key silently reusing the old download is the only real footgun here, and tying the key to the strings it describes makes drift impossible to overlook.

## Replacing the startscript.txt

The start script was generated by BAR from running the specific benchmark scenarios in the actual BAR lobby. Then the startscript was pulled from the bar data dir. Should the startscript ever need changing, that's the easiest way to generate it.

An additional "2025:quitforce" needs to be appended to the end of the start script's debugcommands so that the engine actually quits when finished.
